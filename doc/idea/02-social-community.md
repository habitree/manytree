## 조사 배경

### 왜 소셜/커뮤니티인가?

- **네트워크 효과**: 사용자가 늘수록 앱의 가치가 증가
- **사회적 책임감**: 혼자보다 함께할 때 습관 유지율 향상
- **콘텐츠 생성**: 사용자가 만드는 리뷰, 추천이 핵심 자산
- **입소문 마케팅**: 바이럴 성장의 핵심 동력

### 핵심 원칙

1. **가벼운 연결부터**: 팔로우 → 댓글 → 클럽 순서로 점진적 도입
2. **프라이버시 존중**: 독서는 개인적 활동, 공개 수준 선택권
3. **긍정적 환경**: 비교/경쟁보다 격려/공유 중심

## 벤치마킹 분석

### 1. Goodreads

| 기능 | 장점 | 단점 |
|------|------|------|
| 친구 피드 | 친구 독서 근황 확인 | 피드 알고리즘 부재 |
| 책 리뷰 | 방대한 리뷰 데이터 | 스팸/저품질 리뷰 |
| 독서 챌린지 | 연간 목표 설정 | 개인 목표만 가능 |
| 그룹 | 관심사별 커뮤니티 | 활성도 낮음 |

**핵심 인사이트**: 풍부한 기능이 있지만 UX가 구식, 활성화 어려움

### 2. Instagram

| 기능 | 효과 |
|------|------|
| 스토리 | 일상 공유, 낮은 진입장벽 |
| 피드 | 큐레이션된 콘텐츠 |
| 릴스 | 짧은 영상으로 확산 |
| DM | 1:1 소통 |

**핵심 인사이트**: 비주얼 중심, 스토리의 휘발성이 참여 유도

### 3. Strava (운동 앱)

| 기능 | 효과 |
|------|------|
| 활동 피드 | 운동 기록 공유 |
| 쿠도스 (Kudos) | 간단한 응원 (좋아요) |
| 세그먼트 | 특정 구간 경쟁 |
| 클럽 | 지역/관심사 그룹 |

**핵심 인사이트**: "운동"을 "독서"로 바꾸면 거의 그대로 적용 가능

### 4. 밀리의서재

| 기능 | 효과 |
|------|------|
| 서재 공유 | 독서 취향 표현 |
| 밀리 메이트 | 큐레이터 시스템 |
| 읽고 싶은 책 | 위시리스트 공유 |

**핵심 인사이트**: 한국 시장 맞춤형 소셜 기능

## 기능 아이디어

### 1. 팔로우/팔로워 시스템

#### 기본 구조
```
사용자 프로필
├── 팔로잉 수
├── 팔로워 수
├── 완독 책 수
├── 공개 서재
└── 최근 활동
```

#### 프라이버시 설정
| 설정 | 설명 |
|------|------|
| 전체 공개 | 모든 사용자가 프로필 확인 가능 |
| 팔로워만 | 팔로워만 상세 정보 확인 |
| 비공개 | 프로필 숨김 (팔로우 요청 승인 필요) |

#### 친구 찾기
- 연락처 동기화 (선택)
- 카카오/네이버 친구 연결
- 사용자 검색 (닉네임, 관심 장르)
- 추천 사용자 (비슷한 취향)

### 2. 커뮤니티 피드

#### 피드 유형
```
홈 피드
├── 팔로잉 활동
│   ├── 책 완독
│   ├── 리뷰 작성
│   ├── 위시리스트 추가
│   └── 배지 획득
├── 인기 게시물
└── 추천 사용자
```

#### 게시물 형태
| 유형 | 자동 생성 | 사용자 작성 |
|------|----------|------------|
| 책 완독 | O | - |
| 독서 리뷰 | - | O |
| 책 추천 | - | O |
| 명언/구절 | - | O |
| 스트릭 달성 | O | - |
| 배지 획득 | O | - |

#### 인터랙션
- 좋아요 (읽을래요)
- 댓글
- 책갈피 (저장)
- 공유 (외부 SNS)

### 3. 독서 클럽

#### 클럽 유형
```
공개 클럽
├── 누구나 가입 가능
├── 검색에 노출
└── 최대 멤버: 1,000명

비공개 클럽
├── 초대로만 가입
├── 검색에 비노출
└── 최대 멤버: 50명

1:1 함께 읽기
├── 친구와 같은 책 읽기
└── 진행률 공유
```

#### 클럽 활동
- 공동 독서 목표 설정
- 클럽 전용 토론방
- 멤버별 진행률 확인
- 클럽 랭킹/리워드

### 4. 함께 읽기 챌린지

#### 챌린지 유형
| 유형 | 설명 | 예시 |
|------|------|------|
| 공식 챌린지 | ReadingTree 주관 | 새해 독서 마라톤 |
| 유저 챌린지 | 사용자 생성 | 7월 SF 읽기 |
| 테마 챌린지 | 시즌별 | 여름 휴가 추천도서 |

#### 챌린지 구조
```
챌린지 정보
├── 기간 (시작~종료)
├── 목표 (권수 or 페이지 or 시간)
├── 참여자 현황
├── 리더보드
└── 완료 보상 (배지, XP)
```

### 5. 소셜 공유 기능

#### 공유 가능 항목
- 나의 독서 나무
- 완독 카드
- 연간 독서 리포트
- 스트릭 기록
- 특별 배지

#### 공유 채널
- 카카오톡
- 인스타그램 스토리
- 트위터/X
- 페이스북
- 링크 복사

#### 공유 카드 디자인
```
┌─────────────────────────┐
│    [나무 이미지]         │
│                         │
│  @username의 독서 나무    │
│  📚 42권 읽음            │
│  🔥 15일 연속 독서        │
│                         │
│  ReadingTree로 함께해요!  │
└─────────────────────────┘
```

### 6. 독서 숲 (Forest)

#### 개념
- 내 나무 + 친구들 나무 = 독서 숲
- 숲이 클수록 네트워크 효과

#### 숲 시각화
```
개인 숲
├── 나의 나무 (중앙)
├── 친구 나무들 (주변)
├── 숲 크기 = 팔로워 수
└── 숲 건강도 = 전체 활동량

클럽 숲
├── 멤버들의 나무 모음
├── 클럽 공동 목표 나무
└── 시즌별 특별 테마
```

## ReadingTree 적용 방안

### Phase 1: 기본 소셜 (2-3개월)

1. **팔로우 시스템**
   - 팔로우/팔로워 기능
   - 프로필 페이지
   - 프라이버시 설정

2. **기본 피드**
   - 팔로잉 활동 피드
   - 좋아요 기능

### Phase 2: 인터랙션 강화 (3-4개월)

1. **댓글 & 공유**
   - 게시물 댓글
   - 외부 SNS 공유
   - 공유 카드 생성

2. **함께 읽기**
   - 1:1 함께 읽기
   - 간단한 챌린지

### Phase 3: 커뮤니티 (4-6개월)

1. **독서 클럽**
   - 클럽 생성/가입
   - 클럽 피드
   - 클럽 챌린지

2. **독서 숲**
   - 친구 숲 시각화
   - 클럽 숲

## 예상 효과 및 리스크

### 예상 효과

| 지표 | 예상 변화 |
|------|----------|
| 유저 획득 (신규) | +50% (바이럴) |
| 체류 시간 | +40% |
| Day 30 리텐션 | +15% |
| 콘텐츠 생성량 | +200% |

### 잠재적 리스크

| 리스크 | 영향도 | 대응 방안 |
|--------|--------|----------|
| 부정적 댓글/분위기 | 높음 | 신고 시스템, AI 필터링 |
| 프라이버시 우려 | 중간 | 세분화된 공개 설정 |
| 비교 스트레스 | 중간 | 격려 중심 UX, 랭킹 숨김 옵션 |
| 스팸/광고 | 중간 | 게시물 검토, 신고 기능 |

### 성공 지표

- 평균 팔로잉 수: 5명 이상
- 주간 피드 조회: DAU의 60%
- 독서 클럽 가입률: MAU의 20%
- 외부 공유 횟수: 월 5,000회
