# 컨텐츠 제작 가이드라인

## 1. 테스트 구조

### 1.1 기본 구성

```
테스트 구조
├── 인트로 (소개 페이지)
├── 질문 (8-12개)
├── 로딩 (분석 중)
└── 결과 (유형 + 설명)
```

### 1.2 권장 사양

| 항목 | 권장 | 비고 |
|------|------|------|
| 질문 수 | 8-12개 | 너무 적으면 신뢰도↓, 많으면 이탈↑ |
| 선택지 수 | 2-4개 | 3-4개 권장 |
| 결과 유형 수 | 4-16개 | 4, 8, 12, 16 추천 |
| 완료 시간 | 2-4분 | 3분 이내 최적 |

## 2. 질문 설계

### 2.1 질문 유형

#### A. 상황 제시형 (권장)
```
상황: "친구가 갑자기 고민을 털어놓는다면?"

A. 일단 끝까지 들어준다
B. 해결책을 제시한다
C. 나도 비슷한 경험을 공유한다
D. 기분 전환을 위해 다른 얘기를 한다
```

#### B. 직접 질문형
```
질문: "스트레스를 받을 때 나는?"

A. 혼자만의 시간이 필요하다
B. 누군가에게 이야기한다
C. 운동이나 활동을 한다
D. 맛있는 걸 먹는다
```

#### C. 양자택일형
```
질문: "나는 ___에 더 가깝다"

A. 계획적인 편 ←→ B. 즉흥적인 편
```

### 2.2 질문 작성 원칙

**DO (권장)**
- 일상적인 상황 사용
- 모든 선택지가 매력적이도록
- 정답이 없는 질문
- 짧고 명확한 문장

**DON'T (지양)**
- 너무 추상적인 질문
- 사회적으로 바람직한 답이 뻔한 질문
- 부정적인 표현
- 전문 용어 사용

### 2.3 선택지 설계

```
좋은 예:
Q. 주말 아침, 알람 없이 눈을 떴다면?
A. 그래도 일찍 일어나 계획대로 움직인다 [계획형]
B. 좀 더 누워있다가 배고프면 일어난다 [본능형]
C. SNS나 뉴스부터 확인한다 [정보형]
D. 창밖을 보며 오늘 뭐 할지 생각한다 [사색형]

나쁜 예:
Q. 당신은 계획적인 사람입니까?
A. 매우 그렇다 [정답처럼 느껴짐]
B. 그렇다
C. 아니다
D. 전혀 아니다 [부정적으로 느껴짐]
```

## 3. 결과 설계

### 3.1 결과 구성 요소

```json
{
  "id": "result-type-a",
  "title": "감성적인 힐러형",           // 유형 이름 (인상적으로)
  "subtitle": "따뜻한 공감 능력의 소유자", // 부제목
  "emoji": "🌸",                        // 대표 이모지
  "description": "...",                 // 상세 설명 (3-4문장)
  "traits": ["공감 능력", "섬세함"],    // 핵심 특성
  "strengths": ["..."],                 // 강점
  "tips": ["..."],                      // 조언/팁
  "compatibility": {                    // 궁합 (선택)
    "best": "논리적인 분석가형",
    "worst": "냉철한 비평가형"
  }
}
```

### 3.2 결과 작성 원칙

**1. 긍정적 프레이밍**
```
❌ "당신은 우유부단한 편입니다"
✅ "당신은 신중하게 결정하는 타입입니다"

❌ "결단력이 부족합니다"
✅ "여러 가능성을 열어두는 유연함이 있습니다"
```

**2. 구체적인 예시 포함**
```
❌ "당신은 창의적입니다"
✅ "문제가 생기면 남들이 생각하지 못한 방법을 찾아냅니다.
    일상의 작은 것에서도 새로운 아이디어를 발견하죠."
```

**3. 공감 포인트 삽입**
```
"가끔 너무 많이 생각해서 지칠 때도 있지 않나요?
 그 섬세함이 당신의 강점이지만, 적당한 휴식도 필요해요."
```

**4. 실용적 조언 제공**
```
💡 이런 점을 활용해보세요:
- 브레인스토밍이 필요한 회의에서 의견을 내보세요
- 창작 활동으로 에너지를 발산해보세요
```

### 3.3 결과 유형 네이밍

**패턴 1: 형용사 + 명사**
- 감성적인 힐러
- 논리적인 전략가
- 자유로운 탐험가

**패턴 2: 동물/캐릭터 비유**
- 신중한 부엉이
- 열정적인 사자
- 느긋한 판다

**패턴 3: 별명/밈 스타일 (Z세대)**
- 인간 계획표
- 걸어다니는 백과사전
- 텐션 요정

## 4. 점수 계산 로직

### 4.1 유형 분류 방식

#### A. 축(Axis) 기반 (MBTI 스타일)
```
축1: 외향(E) ←→ 내향(I)
축2: 감각(S) ←→ 직관(N)
축3: 사고(T) ←→ 감정(F)
축4: 판단(J) ←→ 인식(P)

→ 총 16가지 조합 (2^4)
```

#### B. 점수 합산 방식
```
각 유형별 점수 합산 → 최고점 유형 = 결과

예:
- A유형 점수: 15점
- B유형 점수: 22점 ← 최고점
- C유형 점수: 18점
→ 결과: B유형
```

#### C. 가중치 적용 방식
```
핵심 질문에 높은 가중치 부여

Q1(핵심): A선택 → A유형 +3점
Q5(보조): A선택 → A유형 +1점
```

### 4.2 권장 로직

| 결과 유형 수 | 권장 방식 |
|-------------|----------|
| 4개 | 축 1개 (2x2) 또는 점수 합산 |
| 8개 | 축 3개 (2x2x2) |
| 12개 | 점수 합산 + 세부 분류 |
| 16개 | 축 4개 (MBTI 스타일) |

## 5. 톤앤매너

### 5.1 전체 톤

```
✅ 친근하고 따뜻한
✅ 가볍지만 신뢰감 있는
✅ 재미있지만 의미 있는
✅ 공감하는 친구처럼

❌ 딱딱하고 학술적인
❌ 너무 가볍고 진지하지 않은
❌ 판단하거나 평가하는
❌ 부정적이거나 비관적인
```

### 5.2 문체 예시

**인트로**
```
"바쁜 일상 속, 잠시 멈추고 나를 돌아볼 시간이에요.
12가지 질문으로 당신의 숨겨진 모습을 찾아볼까요?"
```

**질문**
```
"카페에서 친구를 기다리는 동안, 당신은?"
```

**로딩**
```
"당신의 답변을 분석하고 있어요...🔍"
```

**결과**
```
"당신은 [따뜻한 공감러] 타입이에요!

주변 사람들의 감정을 누구보다 잘 읽어내는 당신.
말하지 않아도 상대방이 무엇을 원하는지 알죠.
그래서 사람들은 당신 곁에 있으면 편안함을 느껴요."
```

## 6. 품질 체크리스트

### 제작 완료 전 확인

**질문**
- [ ] 모든 질문이 명확하고 이해하기 쉬운가?
- [ ] 선택지가 서로 구분되는가?
- [ ] 정답처럼 느껴지는 선택지가 없는가?
- [ ] 질문 수가 8-12개인가?

**결과**
- [ ] 모든 결과가 긍정적으로 프레이밍되었는가?
- [ ] 구체적인 예시와 조언이 포함되어 있는가?
- [ ] 유형 이름이 매력적인가?
- [ ] 공유하고 싶은 결과인가?

**전체**
- [ ] 완료까지 3분 이내인가?
- [ ] 톤앤매너가 일관적인가?
- [ ] 맞춤법/오타 확인했는가?
- [ ] 모바일에서 잘 보이는가?
